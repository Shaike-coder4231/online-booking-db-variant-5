# Онлайн-запись в салон красоты с выбором мастера и дополнительных услуг

**Выполнил:** Баженов Илья
**Группа:** 454
**Вариант:** 5
**СУБД:** MySQL 8.0+ | **DAL:** PHP 8.x (PDO)
**Кодировка:** utf8mb4_unicode_ci

## Описание предметной области

Система предназначена для автоматизации процесса онлайн-записи клиентов в салон красоты. Реализована возможность выбора конкретного мастера, дополнительных услуг и расходных материалов. Некоторые услуги требуют обязательной предварительной консультации. При создании записи система проверяет наличие необходимых продуктов на складе.

## Стек технологий

- СУБД: MySQL 8.0 или выше
- Уровень доступа к данным: PHP 8.x, PDO, паттерн Repository
- Графические клиенты: MySQL Workbench / DBeaver / DataGrip
- Документация: Markdown
- Контроль версий: Git + GitHub

## Структура репозитория

В корне проекта находится файл README с описанием работы и инструкциями. Конфигурация подключения вынесена в config.example.php, а сам config.php игнорируется системой контроля версий для безопасности. Скрипт demo.php демонстрирует работу уровня доступа к данным. Папка sql содержит скрипт создания схемы schema.sql и файл проверки ограничений constraints_test.sql. В директории docs хранится отчёт по проектированию report.md, отчёт по уровню доступа dal_report.md и подпапка images для скриншотов. Папка queries включает отдельные файлы с аналитическими запросами. В diagrams размещена ER-диаграмма er_diagram.png. Исходный код DAL расположен в папке src и содержит классы Database, RepositoryException, AbstractRepository и специализированные репозитории для каждой сущности. Файл .gitignore настроен для исключения системных, временных файлов и конфигурации с паролями.

## Инструкция по развёртыванию базы данных

1. Склонируйте репозиторий: `git clone https://github.com/Shaike-coder4321/praktika.git`
2. Перейдите в папку проекта: `cd praktika`
3. Подключитесь к серверу MySQL: `mysql -u root -p`
4. Выполните скрипт создания и заполнения схемы: `mysql -u root -p < sql/schema.sql`
5. База данных online_beauty_salon_variant5 будет создана, проиндексирована и заполнена тестовыми данными.
6. Для проверки ограничений выполните: `mysql -u root -p < sql/constraints_test.sql`

## 🐘 Уровень доступа к данным (Data Access Layer)

Реализован на PHP с использованием PDO и паттерна Repository. Код инкапсулирует SQL-запросы, обеспечивает защиту от инъекций через подготовленные выражения и управляет транзакциями при сложных операциях.

### Установка и настройка
1. Создайте файл конфигурации: `cp config.example.php config.php`
2. Откройте config.php и укажите реальные параметры подключения к вашей локальной БД (хост, пользователь, пароль).
3. Убедитесь, что установлен PHP 8.0+ и расширение PDO MySQL.

### Запуск демонстрационного скрипта
Выполните в терминале: `php demo.php`
Скрипт последовательно демонстрирует: подключение к БД, выборку клиентов и мастеров, фильтрацию услуг, создание записи с транзакцией, изменение статуса, удаление и проверку защиты от SQL-инъекций. Результаты выводятся в консоль в читаемом формате.

### Архитектура DAL
- Database.php управляет единственным соединением через Singleton
- AbstractRepository.php содержит базовые CRUD-методы с безопасной динамической сортировкой
- Специализированные репозитории реализуют бизнес-методы (проверка слотов, учёт консультаций, расчёт остатков)
- Все операции с несколькими таблицами обернуты в транзакции с откатом при ошибках
- Подробное описание архитектуры и мер безопасности: docs/dal_report.md
